## Как всё работает

1. Клиент открывает бота в Telegram → нажимает «💅 Открыть каталог услуг»
2. Открывается Mini App (GitHub Pages) → выбирает услугу, дату и время → нажимает «Записаться»
3. TMA отправляет данные боту через `tg.sendData()`
4. Бот получает данные через webhook → сохраняет запись в БД → уведомляет мастера и клиента
5. Мастер заходит в **Админку** → вкладка «Записи» → нажимает «Подтвердить» или «Отменить»
6. Клиент получает уведомление в Telegram о статусе записи

## Как обновить код на сервере

Сервер обновляется вручную через Python + paramiko (SSH).

Файл деплоя: `C:/Users/79054/deploy.py` (telegram.py)
Файл деплоя: `C:/Users/79054/deploy_admin.py` (admin.py)

Команды на сервере:
```bash
# Перезапустить сервис
systemctl restart ma-master

# Посмотреть логи
journalctl -u ma-master -n 50 --no-pager

# Проверить статус
systemctl status ma-master
```
